# 🚀 Onboarding Guide for Shveyka MES

Этот документ — инструкция для **нового AI-агента или разработчика**, который присоединяется к проекту.

---

## 👋 Приветствие

Привет! Ты работаешь над **Shveyka MES** — системой управления производством для швейной фабрики.
Проект состоит из двух Next.js 15 приложений (`crm` и `worker-app`) с базой данных **Supabase (PostgreSQL)**.

---

## 📚 ЧТО ТЕБЕ НУЖНО ЗНАТЬ (Список литературы)

Перед тем как писать код, **ОБЯЗАТЕЛЬНО** изучи эти файлы (в таком порядке):

1.  **`docs/README.md`** — Навигация и статус всей документации.
2.  **`docs/architecture/overview.md`** — Общая схема системы, Data Flow и Clean Architecture.
3.  **`docs/schemas/database.md`** — Полная схема БД. **Не выдумывай колонки!** Смотри сюда.
4.  **`docs/glossary/terms.md`** — Словарь терминов и ролей (что такое "партия", "настил", "Мастер").
5.  **`DESIGN.md`** — Дизайн-система (Linear Style). Мы используем **только CSS-переменные** (например, `var(--bg-base)`). **Запрещено:** использовать `dark:` utility-классы Tailwind.
6.  **`docs/tech-debt.md`** — Список текущего техдолга и планов (например, Telegram Bot, исправление API зарплаты).

---

## ⚠️ СТРОГИЕ ПРАВИЛА (MUST FOLLOW)

1.  **Документация = Код:** Если ты меняешь код, ты **ОБЯЗАН** обновить документацию (Mermaid схемы, OpenAPI, ADR). Без этого задача не готова. (См. стандарт в `docs/README.md`).
2.  **Архитектура:** Соблюдай принципы **Clean Architecture**. Бизнес-логика отдельно, инфраструктура отдельно. (См. `docs/modules/overview.md`).
3.  **Безопасность:** Любое API действие должно проверять роль пользователя (`getAuth()`).
4.  **Стиль UI:** Все новые интерфейсы делай в стиле Linear (см. `DESIGN.md`).
5.  **Язык:** Интерфейс — на **Украинском**. Комментарии в коде — на **Английском** или **Русском** (по контексту).
