# Навык: Анализ производства (Production Analysis)

Этот файл содержит "Ground Truth" правила для ИИ-ассистента системы "Швейка".

## 1. Приоритеты партий
- **is_urgent = true**: Партия имеет высший приоритет. При задержке более чем на 24 часа — статус "Критично".
- **priority**: Числовое значение (1-5). Чем выше число, тем быстрее партия должна пройти стадию кроя.

## 2. Стадии производства (Workflow)
1. **created**: Зарождение. Если висит более 3 дней без перехода в `cutting` — сигналь о застое.
2. **cutting**: Раскрой. Критическая стадия. 
3. **sewing**: Пошив. Если количество выполненных операций (`operation_entries`) меньше 10% от объема партии спустя 2 дня пошива — это риск.

## 3. Операционные записи (Entries)
- Каждая запись — это прогресс. 
- Если за последние 12 часов нет новых записей (`operation_entries`) для активной партии — это "холодная зона".

## 4. Как формулировать инсайты
- Использовать терминологию: "Партия №...", "Стадия...", "Застой", "Риск срыва".
- **Запрещено**: Делать предположения о причинах (например, "швея заболела"), если данных об этом нет в контексте.
- **Обязательно**: Предлагать следующее действие (например, "рекомендуется перенести ресурсы на партию №...")
