# 📚 Shveyka MES — Глоссарий (Glossary)

> Единый словарь терминов для команды, документации и кода.

## 1. Основные сущности

| Термин (RU) | Термин (UK) | Код (Variable) | Описание |
|-------------|-------------|----------------|----------|
| **Партия** | Партія | `Batch` / `production_batch` | Единица производства. Часть заказа, содержащая конкретное кол-во изделий одной модели, ткани и размера. |
| **Заказ** | Замовлення | `Order` / `production_order` | Документ от клиента или план производства, содержащий одну или несколько партий. |
| **Задача** | Завдання | `Task` / `batch_task` | Поручение конкретному сотруднику/роли на выполнение определенного этапа для партии. |
| **Запись** | Запис | `Entry` / `task_entry` | Факт выполненной работы сотрудником (кто, сколько, когда). Основа для начисления ЗП. |
| **Настил** | Настил | `Nastil` (Legacy) | Специфический термин для этапа Раскрой. Описывает слои ткани, рулоны и длину. |
| **Этап** | Етап | `Stage` / `production_stage` | Технологический шаг процесса (Раскрой, Пошив, Упаковка). |
| **Операция** | Операція | `Operation` | Конкретное действие внутри этапа (например, "Настил", "Крой"), имеющее ставку оплаты. |

## 2. Роли (Roles)

Роли определяются в таблице `users` и фильтруют доступ к задачам в Worker App.

| Код роли | Название | Функция |
|----------|----------|---------|
| `admin` | Администратор | Полный доступ ко всему. |
| `manager` | Менеджер | Управление заказами, партиями, начисление ЗП. |
| `master` | Мастер цеха | Контроль производства, подтверждение этапов (упразднено в пользу Начальника). |
| `production_head` | Начальник пр-ва | Подтверждение выполнения этапов и передача партий дальше. |
| `cutting` | Раскройщик | Выполняет задачи раскроя в Worker App. |
| `sewing` | Швея | Выполняет задачи пошива в Worker App. |
| `overlock` | Оверлочник | Выполняет задачи оверлока. |
| `packaging` | Упаковщик | Выполняет задачи упаковки. |

## 3. Статусы Партии (Batch Status)

Используются в поле `production_batches.status`.

| Статус | Описание |
|--------|----------|
| `created` | Создана, но еще не запущена в работу. |
| `cutting` | На этапе раскроя. |
| `sewing` | На этапе пошива. |
| `overlock` | На этапе оверлока. |
| `straight_stitch` | На этапе прямострочки. |
| `coverlock` | На этапе распошива. |
| `packaging` | На этапе упаковки. |
| `ready` | Готова к отгрузке. |
| `shipped` | Отгружена клиенту. |
| `cancelled` | Отменена. |

## 4. Статусы Задачи (Task Status)

Используются в поле `batch_tasks.status`.

| Статус | Описание |
|--------|----------|
| `pending` | Ожидает, пока сотрудник возьмет её в работу. |
| `accepted` | Сотрудник принял задачу. |
| `in_progress` | В процессе выполнения (введено минимум одно действие). |
| `completed` | Сотрудник нажал "Готово". Ждет подтверждения Начальника. |

## 5. Статусы Записи (Entry Status)

Используются в поле `task_entries.status`.

| Статус | Описание |
|--------|----------|
| `submitted` | Запись создана работником. Ожидает проверки (ЗП не начислена). |
| `approved` | Запись подтверждена Начальником. ЗП начислена. |
| `rejected` | Запись отклонена (брак, пересорт). |
