# Бізнес-процес: Створення виробничого замовлення та партій

Цей документ фіксує етапи роботи Начальника виробництва в системі MES.

## Етап 1: Створення нового замовлення
1. **Дія**: Натискання кнопки "Створити замовлення".
2. **Форма (Модальне вікно)**:
    - **Куди**: Вибір (Склад / Замовник).
    - **Базова модель**: Вибір із довідника конструктів (напр. "Футболка для хлопчика").
    - **Кількість (план)**: Загальна планова кількість одиниць для всього замовлення.
3. **Результат**: Створення замовлення.
4. **Статус**: `Нове`.

## Етап 2: Запуск у виробництво
1. **Дія**: Перехід у замовлення та натискання "Запустити у виробництво".
2. **Мета**: Підтвердження наміру виробництва та перехід до формування конкретних завдань для цеху розкрою.

## Етап 3: Формування виробничих партій (Розкрій)
1. **Дія**: Натискання "Створити партію" всередині активного замовлення.
2. **Логіка вибору розмірів**:
    - Система підтягує розмірну сітку базової моделі.
    - Начальник виробництва **обирає (фіксує)** перелік необхідних розмірів.
    - Кількість штук по кожному конкретному розміру **не вказується**.
3. **Параметри матеріалу (Колір / Рулони / Кількість)**:
    - Начальник додає одну або кілька позицій тканини.
    - Для кожної позиції вказується:
        - **Колір тканини** (вибір із доступних).
        - **Кількість рулонів** (скільки фізично взяти зі складу).
        - **Планова кількість штук** (скільки виробів цього кольору планується отримати).
4. **Результат**: Створення партії, що містить обрану сітку розмірів та деталізацію за кольорами/рулонами.
5. **Статус партії**: `Нова`.

## Етап 4: Передача на розкрій
1. **Системна дія**: Надсилання Push-повідомлення.
2. **Отримувач**: Співробітники з роллю "Розкрій".
3. **Відображення**: Партія з'являється в інтерфейсі "Розкрій" мобільного додатка Worker.
