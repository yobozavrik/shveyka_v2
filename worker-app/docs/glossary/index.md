# Глосарій — МЕС ЦЕХ Worker App

> Офіційний словник термінів проекту. Всі учасники повинні використовувати ці терміни однаково.

---

## Сущності

| Термін | Визначення | Код/Таблиця |
|--------|-----------|-------------|
| **Партія** | Одиниця виробництва: конкретне замовлення на пошиття моделі з певною кількістю, тканиною, кольорами | `production_batches` |
| **Номер партії** | Унікальний ідентифікатор партії у форматі `П-{номер}` (напр. П-234) | `batch_number` |
| **Модель** | Виріб, який шиється (футболка, худі, брюки) | `product_models` |
| **Розмірна сітка** | Набір розмірів для партії (S, M, L, XL, XXL) | `size_variants` (JSONB) |
| **Етап** | Стадія виробництва (розкрой, пошив, упаковка) | `production_stages` |
| **Операція** | Конкретна дія всередині етапу з формою вводу | `stage_operations` |
| **Завдання** | Прив'язка працівника до партії + етапу. Має статус | `batch_tasks` |
| **Запис** | Факт виконаної роботи: кількість, дані форми, час | `task_entries` |
| **Настил** | Один рулон тканини, розкладений для розкрою | `cutting_nastils` |
| **quantity_per_nastil** | Кількість деталей НА ОДИН РОЗМІР з одного настилу | `task_entries.quantity` |
| **Конвеєр** | Послідовність операцій партії від початку до кінця | `route_card_operations` |

## Ролі

| Термін | Визначення |
|--------|-----------|
| **Розкройщик** | Працівник з роллю `cutting`. Працює з настилами та кроєм |
| **Швець** | Працівник з роллю `sewing`. Виконує пошив |
| **Оверлочник** | Працівник з роллю `overlock` |
| **Прямострочник** | Працівник з роллю `straight_stitch` |
| **Распошівник** | Працівник з роллю `coverlock` |
| **Упаковщик** | Працівник з роллю `packaging` |
| **Майстер** | Керівник зміни. Роль `master`. Підтверджує записи, призначає техпроцеси |
| **Адміністратор** | Повний доступ. Роль `admin` |

## Статуси задач

| Статус | Значення | Коли встановлюється |
|--------|---------|-------------------|
| `pending` | Очікує прийняття | При створенні задачі |
| `accepted` | Працівник прийняв | Після натискання "Прийняти в роботу" |
| `in_progress` | В процесі виконання | Після першого запису (автоматично) |
| `completed` | Завершено | Після натискання "Завершити етап" |
| `cancelled` | Скасовано | Адміністратором |

## Бізнес-терміни

| Термін | Визначення |
|--------|-----------|
| **Piecework** | Відрядна оплата: працівник отримує за кожну одиницю продукції |
| **Payroll period** | Період нарахування зарплати (тиждень/місяць) |
| **Брак** | Деталь з дефектом. Вираховується із загальної кількості |
| **Якість** | Відсоток якісних деталей: `(total - defect) / total × 100` |
| **Ціль дня** | Рекомендована кількість штук на день (avg × 1.2) |
| **КТ** | Контрольна точка. Операція, що вимагає перевірки майстром |
| **Техпроцес** | Набір операцій для партії (route card) |

## Технічні терміни

| Термін | Визначення |
|--------|-----------|
| **CRM** | Основна система управління виробництвом (окремий проект) |
| **Worker App** | Мобільний додаток працівників (цей проект) |
| **Supabase** | PostgreSQL + auth + realtime. База даних проекту |
| **Schema `shveyka`** | Схема БД з виробничими таблицями |
| **Schema `public`** | Схема БД з користувачами, audit логами |
| **JWT** | JSON Web Token. Cookie `mes_worker_token` для auth |
| **N+1 query** | Антипатерн: N окремих запитів замість одного з IN() |
| **field_schema** | JSONB-схема полів форми операції |
